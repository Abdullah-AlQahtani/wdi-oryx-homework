    <article class="markdown-body entry-content" itemprop="text"><h1><a id="user-content-wdi-1-general-assembly-riyadh" class="anchor" aria-hidden="true" href="#wdi-1-general-assembly-riyadh"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>WDi-1 (<a href="https://generalassemb.ly/riyadh" rel="nofollow">General Assembly, Riyadh</a>)</h1>

<h3><a id="user-content-1-git-config" class="anchor" aria-hidden="true" href="#1-git-config"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>1. Git Config</h3>
<p>You only need to do this part once on your machine.</p>
<div class="highlight highlight-source-shell"><pre>git config --global user.name <span class="pl-s"><span class="pl-pds">"</span>ReplaceMe<span class="pl-pds">"</span></span>
git config --global user.email <span class="pl-s"><span class="pl-pds">"</span>replace@me.com<span class="pl-pds">"</span></span>

<h3><a id="user-content-2-repository-setup" class="anchor" aria-hidden="true" href="#2-repository-setup"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>2. Repository Setup</h3>
<p>You only need to do this part once, not every time you're submitting homework!</p>
<ul>
<li><strong>Fork this repository</strong>
<ul>
<li><em>'Forking' creates a personal, 'forked' copy of this repository on your Github account.</em></li>
<li>Hit the <strong>Fork</strong> button in the top right-hand corner of this page.</li>
</ul>
</li>
<li><strong>Clone your forked repository to your computer</strong>
<ul>
<li><em>'Cloning' takes your 'forked' repository on GitHub and creates a local copy (aka 'clone') on your computer.</em></li>
<li>Make sure you're browser is open to <strong>your</strong> forked version of this repository on Github (e.g. <a href="http://github.com/%7B%7BYOUR_USERNAME%7D%7D/wdi-oryx-homework">http://github.com/{{YOUR_USERNAME}}/wdi-oryx-homework</a>).</li>
<li>Hit the <strong>Clone or Download</strong> button in the top right-hand corner of the page and copy the URL to your clipboard.</li>
<li>Open Terminal/iTerm2 to the directory in which you intend to store your homework.</li>
<li><code>git clone url_of_your_fork_on_github</code> (where <code>url_of_your_fork_on_github</code> is the URL you copied after hitting 'Clone or Download', above).</li>
</ul>
</li>
<li><strong>Add an upstream remote repository</strong>
<ul>
<li><em>Creating an upstream remote links the local repository on your computer to this repository on Github</em></li>
<li><code>cd wdi-oryx-homework</code></li>
<li><code>git remote add upstream https://github.com/wdi-oryx/wdi-oryx-homework</code></li>
<li><code>git pull upstream master</code></li>
</ul>
</li>
</ul>
<h3><a id="user-content-3-do-your-homework" class="anchor" aria-hidden="true" href="#3-do-your-homework"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>3. Do your Homework</h3>
<p>You should put each night's homework in a new folder within the appropriate directory of your homework repo. So, for day two, where you have two tasks ("Calculator" and "Strings"), you might do something like this:</p>
<ol>
<li>Open Terminal/iTerm2;</li>
<li>Go to your local homework repo (e.g., <code>cd Projects/general-assembly/wdi-oryx-homework</code>, or wherever your local repo is);</li>
<li>From here, go to the folder matching your name within that repo, and the appropriate week (e.g., <code>Groucho_Marx/week_01</code>);</li>
<li>Create new folders for each of the day's homework tasks: (e.g. <code>mkdir calculator</code> and <code>mkdir strings</code>);</li>
<li>Create the files necessary to complete the homework in their respective directories;</li>
<li>Get to it!</li>
</ol>
<h3><a id="user-content-4-submit-your-homework" class="anchor" aria-hidden="true" href="#4-submit-your-homework"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>4. Submit Your Homework</h3>
<p>You need to do this every time you're submitting homework.</p>
<ul>
<li><strong>Commit your work to your local repository progressively</strong>
<ul>
<li><code>git add .</code></li>
<li><code>git commit -m "your commit message goes here"</code>(where <code>your commit message goes here</code> is your description of the work you are committing, e.g. "added subtraction to JS calculator").</li>
</ul>
</li>
<li><strong>Merge changes from this repository into your own local repository</strong>
<ul>
<li><code>git pull upstream master</code></li>
</ul>
</li>
<li><strong>Push the changes in your local repository to your forked repository</strong>
<ul>
<li><code>git push origin master</code></li>
</ul>
</li>
<li><strong>Once you're finished, submit a pull request for me to accept your homework</strong>
<ul>
<li>Navigate to your forked version of this repository on Github (e.g. <a href="https://github.com/%7B%7BYOUR_USERNAME_HERE%7D%7D/wdi-oryx-homework">https://github.com/{{YOUR_USERNAME_HERE}}/wdi-oryx-homework</a>).</li>
<li>Hit the <strong>Pull request</strong> button.</li>
<li>Make sure the destination for the pull request is set to my repository, not your own or anyone else's.</li>
</ul>
<ul>
<li>In the pull request comment, tell me the following:
<ul>
<li>How difficult did you find this (out of 10)? 0 being no problems at all, 10 being impossible;</li>
<li>Was there anything that you struggled with?;</li>
<li>Is there anything that you'd like some further information on?;</li>
<li>Roughly how long did it take?</li>
</ul>
</li>
</ul>
</li>
</ul>
<p>If you don't mention anything in the Pull Request comments, we will assume you had no problems at all with it!</p>
